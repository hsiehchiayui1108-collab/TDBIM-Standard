# 專案軀體圖編製規範

本規範定義軀體圖（涵蓋建築、結構、裝修、機電）的產出標準，服務模板、鋼筋、混凝土三大工項。

---

## 設計哲學與仲裁原則
* **簡潔優先（KISS）：** 不過度建模。能合併的欄位合併、不必要的欄位刪除。 寧可少一個欄位，需要時再加；不預先建模未證明必要的概念。
* **格式細節不入架構：** 分隔符、前綴、檔名規範等屬實作層細節，不在此架構規範。 架構只定義語意結構，不定義視覺或格式呈現。
* **閱讀優先【仲裁原則】：** 命名與結構服務於讀者理解，不服務於工具或檔案系統的整齊性。 當其他原則衝突時，以本原則為仲裁依據。
* **不擬人化：** 圖紙、視圖、元素是媒介，不是主體。描述時避免使用「回答」「解決」 等擬人化動詞，改用「針對」「呈現」等中性描述詞。
* **單一真相來源（SSOT）：** 每個資訊只在一個地方維護，避免重複。需要引用其他區塊的內容時， 明示引用來源（例「依必要觸發規則.必要性取值」），不重述內容。

## 視圖雙維度結構
視圖由「視圖類型」與「資訊範圍」兩個獨立維度正交組合（索引視圖、資訊區塊、表格不套用此矩陣）：

| 資訊範圍 → <br> 視圖類型 ↓ | 整合 (跨專業界面) | 軀體 (僅含混凝土本體) |
| :--- | :--- | :--- |
| **平面視圖** | 整合平面圖 | 軀體見上圖、軀體見下圖 |
| **剖面視圖** | 整合剖面圖、坡段斜率展開圖 | 軀體剖面圖 |
| **3D 視圖** | 3D 整合圖 | 3D 軀體圖 |

## 元素類別與屬性規範
| 元素類別 | 定義與判別問題 | 強制屬性 / 可選屬性 | 用例 |
| :--- | :--- | :--- | :--- |
| **實體** | 代表施工實際涉及的構件或構造特徵，具有幾何形狀<br>*這是施工會實際做出來、或影響到的物理事物嗎？* | 強制：邊界線<br>可選：切割面填充色, 表面填充色, 填充線 | RC柱、RC梁、RC版、RC牆、輕隔間、門窗牆面開口、樓板開口、RC設備基座 |
| **尺寸標註** | 以數值標示物件之間或物件本身的尺寸關係，由尺寸線與數字構成<br>*這是用數字標出的距離或大小嗎？* | 強制：尺寸線<br>可選：無 | 網格到放樣基準線、柱邊到梁邊、梁邊到梁邊、牆厚、開口定位及尺寸 |
| **符號** | 自由放置的視覺記號或文字標籤，透過形式或位置傳達非尺寸性資訊<br>*這是獨立放置的視覺記號，既不是表單欄位也不是純尺寸數字嗎？* | 強制：線條<br>可選：無 | 修訂符號、雲形線、疑義編號標籤、高低版符號、空間名稱、RC柱編號及斷面尺寸、BIM中心logo |
| **資料欄位** | 存在於結構化容器（標題欄、表單、明細表）中，以「鍵-值」對呈現的欄位。 鍵由容器結構定義，值隨專案填入。<br>*這是表單格子裡有固定鍵、內容可變的資料嗎？* | 強制：<br>可選：無 | 業主名稱、專案名稱、圖號、圖名、版次、疑義編號、提出日期 |

## 目的描述撰寫範本（三段結構）
任何層級的目的描述均須依循以下格式填空：

* **圖紙：** 當 `[使用情境]` 時，提供 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **視圖：** 針對「`[使用情境]`」，以 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **元素（必要）：** 呈現 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **元素（視條件）：** 當 `[使用情境]` 時，呈現 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。

## 命名硬性規則對照表
名稱必須使用連貫自然語言，嚴禁承擔上下文分類責任。

| ❌ 錯誤示範（禁止） | ⭕ 正確示範（推薦） | 違反規則 |
| :--- | :--- | :--- |
| `牆面開口_門窗` | `門窗牆面開口` | 禁止使用下底線等分隔符。應採「[修飾語][類別詞]」。 |
| `（同上）` | `具體元素名稱` | 禁止無語意的省略寫法。 |
| `B1_平面圖` | `平面圖` | 名稱不應包含結構上下文（如樓層）。 |
