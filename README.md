# 專案軀體圖編製規範

本規範定義軀體圖（涵蓋建築、結構、裝修、機電）的產出標準，服務模板、鋼筋、混凝土三大工項。

---

## 設計哲學與仲裁原則
* **最高原則【閱讀優先】：** 命名與結構服務於讀者理解。當原則衝突時，以此為最終仲裁依據。
* **簡潔優先 (KISS)：** 不過度建模，不預先增加未證實必要的欄位。
* **不擬人化：** 媒介非主體。禁用「回答」、「解決」，改用「針對」、「呈現」等中性詞。
* **單一真相來源 (SSOT)：** 資訊不重複維護，引用時明示來源。

---

## 視圖雙維度結構
視圖由「視圖類型」與「資訊範圍」兩個獨立維度正交組合（索引視圖、資訊區塊、表格不套用此矩陣）：

| 資訊範圍 → <br> 視圖類型 ↓ | 整合 (跨專業界面，影響混凝土形狀位置) | 軀體 (僅含混凝土構造本體) |
| :--- | :--- | :--- |
| **平面視圖** | 整合平面圖 | 軀體見上圖、軀體見下圖 |
| **剖面視圖** | 整合剖面圖、坡段斜率展開圖 | 軀體剖面圖 |
| **3D 視圖** | 3D 整合圖 | 3D 軀體圖 |

---

## 目的描述撰寫範本（三段結構）
任何層級的目的描述均須依循以下格式填空：

* **圖紙：** 當 `[使用情境]` 時，提供 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **視圖：** 針對「`[使用情境]`」，以 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **元素（必要）：** 呈現 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。
* **元素（視條件）：** 當 `[使用情境]` 時，呈現 `[核心內容]`，使 `[角色]` 能 `[使用成果]`。

---

## 命名硬性規則對照表
名稱必須使用連貫自然語言，嚴禁承擔上下文分類責任。

| ❌ 錯誤示範（禁止） | ⭕ 正確示範（推薦） | 違反規則 |
| :--- | :--- | :--- |
| `牆面開口_門窗` | `門窗牆面開口` | 禁止使用下底線等分隔符。應採「[修飾語][類別詞]」。 |
| `（同上）` | `具體元素名稱` | 禁止無語意的省略寫法。 |
| `B1_平面圖` | `平面圖` | 名稱不應包含結構上下文（如樓層）。 |
